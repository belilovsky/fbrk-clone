# Исследование контентных сайтов — рекомендации для FBRK

> **Цель:** выявить фичи, которые реализованы на трёх сайтах, но отсутствуют на [FBRK](https://fbrk.qdev.run) — казахстанском изданий о расследовательной журналистике и коррупции.  
> **Дата исследования:** 23 апреля 2026  
> **FBRK уже имеет:** статические страницы, рубрики «Расследования» / «Новости», hero-статья на главной, обложки thumb/web, тёмная тема, мобильная адаптация, поиск-оверлей (Ctrl+K), Telegram-бот как канал анонимных сообщений, ссылка на YouTube-канал в сайдбаре.

## 1. report.qdev.run — QazReport · Отчёты

**Описание:** Аналитическая платформа при QazReport. Предоставляет закрытые аналитические отчёты, мониторинговые панели и инструменты (GDELT/NewsAPI/RSS/Echo Sounder).

### Скриншоты
- Главная: `tool_calls/screenshot/screenshot_report.qdev.run_20260423_191004_mobuw6mx.png`

### Реализованные контентные фичи

| Фича | Статус |
|---|---|
| Поиск | ✗ нет |
| Теги / категории | ✗ нет (это каталог отчётов, не новостной сайт) |
| Связанные материалы | ✗ нет |
| Комментарии | ✗ нет |
| Подписка / рассылка | ✗ нет |
| RSS | упомянут в пункте «Панель мониторинга» — используется как источник данных |
| Социальное шаринг | ✗ нет |
| Тёмная тема | ✗ нет |
| Мобильная адаптация | Базовая |

### UX-паттерны

- Простейший каталог со списком ссылок (≈6 пунктов), нет пагинации и бесконечной прокрутки
- Нет хедера с навигацией, нет футера — минималистичный дизайн «внутреннего инструмента»
- Каждый пункт — заголовок + подзаголовок (период или тип). Никаких карточек, изображений, дат

### Рубрики / категории

Нет традиционной системы категорий. Материалы организованы по типу:
- Аналитические отчёты (12 месяцев, 9 месяцев)
- Методология (S-G Index)
- Панель мониторинга (GDELT/NewsAPI/RSS)
- Инвентарь данных
- Административные инструменты (Echo Sounder)

### Видео / YouTube

Отсутствует.

### Полезные мелочи дизайна

- Карточки с мягкими border-radius на белом фоне, минимальный текст — легко сканируется
- URL-структура указывает на закрытую аналитическую платформу (не публичный контентный сайт)

### Вывод

report.qdev.run — это **внутренний инструмент**, а не публичный медиасайт. Прямых UX-паттернов для адаптации в FBRK практически нет. Ценность: пример того, как структурировать раздел «Методология» или «Архив отчётов» — чистый, лаконичный каталог с минимумом шума.

### UX-паттерны

- **Листинг:** пагинация через кнопку «загрузить ещё» (стрелка вниз), не бесконечная прокрутка
- **Главная:** hero-блок (3 материала в сетке), «Последние новости» — вертикальный список слева + крупные карточки справа, реклама-баннер посередине
- **Категории:** двухуровневая навигация: топ-уровень (Политика, Экономика…) + подкатегории в серой полоске под хедером
- **Статья:** одноколоночный текст + правый сайдбар «Читаемое» (постоянно виден при скролле)
- **404-страница:** с поиском и блоком «Лучшее за неделю» — умная заглушка

### Рубрики / категории

Двухуровневая иерархия:
```
Политика → Внутренняя / Внешняя / Мир / Безопасность / Мнения
Экономика → Госсектор / Финансы / Бизнес / Наука
Общество → События / Жизнь / Религия / Происшествия / Спорт / Техно / Культура
Другое → День в истории / Письма Деду Морозу / ...
Мультимедиа → Фото / Видео / Инфографика
Спецпроект
```

URL-паттерн: `/ru/news/{категория}/{slug}_date_{дата}` — человекочитаемый slug с датой.

### Видео / YouTube

- Отдельный медиа-раздел `Мультимедиа → Видео` с карточками
- На карточках иконка видеокамеры обозначает тип материала
- Видео встроено внутри текста статей (конкретный плеер не определён — может быть YouTube embed)

### Полезные мелочи дизайна

- **Верхняя информ-полоска:** дата, город, температура + режим дня (🌙), курсы трёх валют, телефон, соцсети — всё одним взглядом
- **Иконки типа контента:** маленькая камера/видео прямо на карточке не отвлекает, но информирует
- **Сайдбар «Читаемое»:** sticky-сайдбар на странице листинга и статьи — популярные материалы
- **Кнопка «Предложить новость»:** красная CTA-кнопка в хедере — явная точка вовлечения читателей
- **Социальный шаринг:** расположен сразу под датой и автором, до текста статьи — не в конце

## 4. FBRK (fbrk.qdev.run) — Базовый уровень (для сравнения)

**Текущая структура (из HTML-кода):**

```
Topstrip: дата, Telegram-бот, YouTube, Связаться с редакцией
Masthead: кнопка «Написать в бот», логотип, переключатель темы 🌙, поиск 🔍
Nav: Главная / Расследования / Новости / О нас + поиск
Main:
  - Lead (hero-секция, 1 статья)
  - #investigations (сетка карточек «Расследования»)
  - #latest (список + sidebar)
    Sidebar: виджет Telegram-бот | виджет «Темы» (тег-клауд) | виджет YouTube-канал
Footer: разделы + редакция + анонимно (Telegram) + язык (только Русский)
Search overlay: модальное окно с поиском (Ctrl+K)
```

**Чего нет:**
- Отдельных страниц рубрик (только якоря `#investigations`, `#latest`)
- Страниц отдельных статей (нет slug-роутинга)
- Тегов / хэштегов на статьях
- Счётчика просмотров
- Времени чтения
- Шаринга в соцсетях
- Связанных / похожих статей
- Push-уведомлений
- RSS-ленты
- Пагинации или архива
- Инфографики / фото-галерей
- YouTube-embed на сайте
- Переключателя языка (казахский?)
- Breadcrumbs

## Рекомендации для FBRK

### Приоритет 1 — Must-have (сильно влияют на retention и SEO)

| # | Фича | Обоснование | Пример |
|---|---|---|---|
| 1.1 | **Отдельные URL для статей** (slug-роутинг) | Без собственных URL нельзя шарить статьи, строить SEO, получать обратные ссылки. Сейчас весь контент живёт на `index.html`. | total.kz `/ru/news/{категория}/{slug}_date_{дата}` |
| 1.2 | **Отдельные страницы рубрик** (Расследования, Новости) | Читатель должен уметь смотреть «все расследования» как полноценный листинг, а не скролл-якорь. Нужна пагинация. | total.kz `/ru/news/bezopasnost` |
| 1.3 | **Социальный шаринг на статье** | Кнопки Telegram, X, ВКонтакте под заголовком/в конце статьи. Расследования — материал с высоким sharing-потенциалом. | total.kz: ВКонтакте / Одноклассники / FB / X сразу под датой |
| 1.4 | **Теги / хэштеги на статьях** | Позволяют группировать материалы по персонам (Назарбаев, Масимов), ведомствам, темам. Уже заложен виджет «Темы» в сайдбаре — нужно привязать к страницам тегов. | total.kz: `#статья274УКРК #декриминализация` |
| 1.5 | **Счётчик просмотров** | Сигнал доверия — читатель видит, что материал читают тысячи. Для расследований особенно важен. | total.kz: 👁 936 на карточке |
| 1.6 | **RSS-лента** | Стандарт для медиаиздания. Агрегаторы, боты, другие СМИ подпишутся автоматически. Минимальная реализация — `/feed.xml`. | — |

### Приоритет 2 — Nice-to-have (улучшают UX, engagement)

| # | Фича | Обоснование | Пример |
|---|---|---|---|
| 2.1 | **Время чтения** | Для расследований (длинные тексты) особенно полезно. Рассчитывается на фронте: `Math.ceil(wordCount / 200)` мин. | — |
| 2.2 | **Связанные статьи** («Читайте также») | Снижает bounce rate. Показывать 3–4 материала из той же темы/тега после текста статьи. | total.kz: «Смотрите также» |
| 2.3 | **Топ материалов в сайдбаре** («Читаемое») | Sticky-сайдбар на странице статьи/листинга с популярными материалами за неделю. | total.kz: «Читаемое» |
| 2.4 | **Мультимедиа-раздел** (Видео/Инфографика) | У FBRK уже есть YouTube-канал — надо встроить последние видео прямо на сайт (embed или отдельный раздел). | ortcom.kz: LIVE-раздел с YouTube-embed |
| 2.5 | **Breadcrumbs** | При наличии отдельных страниц рубрик и статей необходимы. Помогают навигации и структурному SEO. | ortcom.kz: «Главная → Новости» |
| 2.6 | **Пресс-анонсы / Ближайшие события** | Виджет с расписанием пресс-конференций / событий редакции (если есть) на главной или в сайдбаре. | ortcom.kz: «Пресс-анонсы — Сегодня / Завтра» |
| 2.7 | **Переключатель языка (КАЗ)** | Казахстанское издание без казахской версии теряет значительную аудиторию. Даже заглушка «скоро» с отдельным URL. | ortcom.kz: КАЗ / РУС / ENG |
| 2.8 | **Умная 404-страница** | Текущая nginx 404 — бренд-урон. Заменить на страницу со ссылками на главные разделы и поиском. | total.kz: 404 с поиском + «Лучшее за неделю» |
| 2.9 | **Фотогалерея в статьях** | Для расследований с документами, схемами, скриншотами — слайдер изображений в теле статьи. | total.kz: фото-раздел, ortcom.kz: галерея в статье |

### Приоритет 3 — Можно позже (дополнительный функционал)

| # | Фича | Обоснование | Пример |
|---|---|---|---|
